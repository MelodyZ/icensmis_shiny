Data Analysis Using R Package icensmis

Yibai Zhao

November 11, 2015

In this section, we would introduce two functions in the package icensmis.

1. Data Simulation

1.1 Function

datasim(N, blambda, testtimes, sensitivity, specificity, betas = NULL, twogroup = NULL, pmiss = 0, pcensor = 0, design = "MCAR", negpred = 1, time.varying = F)

datasim() simulates a longitudinal form data frame of N subjects with misclassified outcomes, assuming each subject receives a sequence of pre-scheduled tests for disease status ascertainment. Each test is subject to error, characterized by sensitivity and specificity. An exponential distribution is assumed for the time to event of interest.

  • N: total number of subjects to be simulated
  • blambda: baseline hazard rate
  • testtimes: a vector of pre-scheduled test times
  • sensitivity: the sensitivity of test
  • specificity: the specificity of test
  • betas: a vector of regression coefficients of the same length as the covariate vector. If betas = NULL then the simulated dataset corresponds to the one sample setting. If betas \(\neq\) NULL and twogroup \(\neq\) NULL then the simulated dataset corresponds to the two group setting, and the first value of betas is used as the coefficient for the treatment group indicator. If betas \(\neq\) NULL and twogroup = NULL, then the covariates are ~ i.i.d. N(0, 1), and the number of covariates is determined by the length of betas.
  • twogroup: corresponds to the proportion of subjects allocated to the baseline (reference) group in the two-group setting. For the two-group setting, this variable should be between 0 and 1. For the one sample and multiple (\(\geq\) 2) covariate setting, this variable should be set to NULL. That is, when betas !=NULL, set twogroup to equal the proportion of the subjects in the baseline group to obtain a simulated dataset corresponding to the two-group setting. Else, set twogroup=NULL to obtain either the one sample setting (betas=NULL) or continuous covariates (betas \(\neq\) NULL).
  • pmiss: a value or a vector (must have same length as testtimes) of the probabilities of each test being randomly missing at each test time. If pmiss is a single value, then each test is assumed to have an identical probability of missingness.
  • pcensor: a value or a vector (must have same length as testtimes) of the probability of censoring at each visit, assuming censoring process is independent on other missing mechanisms.
  • design: missing mechanism: MCAR or NTFP
  • negpred: baseline negative predictive value, i.e.the probability of being truely disease free for those who were tested (reported) as disease free at baseline. If baseline screening test is perfect, then negpred = 1.
  • time.varying: indicator whether fitting a time varying covariate model or not

Additional inputs include the values of the parameters \(\varphi_1\) (sensitivity) and \(\varphi_0\) (specificity) for self-report.

1.2 Example

The structure of the input set is described below:

  • Subject id: a unique id for each subject
  • Visit time: the visit time corresponding to each observation. The number of unique visit times in the data determines the number of parameters (\(\theta\)).
  • Result: the binary diagnostic test/self-report result, where 0 denotes a negative test (absence of event of interest) and 1 denotes a positive test. For self-reported outcomes, we assume that all results after the first positive are non-informative. Thus, all results after the first positive should be excluded from the dataset prior to analysis.
  • Covariates: all explanatory variables needed in the model. If we want to fit a time-varying covariate model, we would need to add an observa- tion for each subject at baseline (time=0) providing baseline covariate values.

2. Data Analysis

2.1 Function

icmis(subject, testtime, result, data, sensitivity, specificity, formula = NULL, negpred = 1, time.varying = F, betai = NULL, initsurv = 0.5, param = 1, ...)

In the function call for the icmis() function, subject denotes subject id, testtime denotes visit time, result denotes binary (0 or 1) self-report, data denotes data frame, sensitivity denotes the value of \(\varphi_1\), specificity denotes the value of \(\varphi_0\) and negpred denotes the baseline negative predictive value (\(\eta\)). The formula argument is used to specify the covariates to be included in the model. Interaction terms can also be included - for example, \(cov1+cov2+cov1*cov2\).

The returned result of the icmis() function includes the value of the log likelihood function at the MLE, estimated regression coefficients , es- timated survival function at each visit time (\(S_2, ..., S_{J+1}\)), and covariance matrix corresponding to the estimated parameters.

2.2 Example

2.2.1 One sample setting

We set betas = NULL and twogroup = NULL corresponding to the one sample setting. See example below:

simdata1 <- datasim(N = 1000, blambda = 0.05, testtimes = 1:8, sensitivity = 0.7,
  specificity = 0.98, betas = NULL, twogroup = NULL, pmiss = 0.3, design = "MCAR")

See function call for icmis() below:

fit1 <- icmis(subject = ID, testtime = testtime, result = result, data = simdata1,
  sensitivity = 0.7, specificity= 0.98, formula = NULL, negpred = 1)

2.2.6 Fit data with time varying covariates

To incorporate time varying covariates, we assume that the covariate values are updated at each visit. Unlike the previous example, the values of the covariates at baseline (time = 0) must be present in the data frame. See example below:

   sensitivity = 0.7, specificity= 0.98, formula = ~cov1+cov2+cov3, negpred = 1,
   time.varying = TRUE)

3. Shiny Application Instructions

Steps
1. Click Choose File to upload a dataset. If you upload successfully, you will see a message below.
2. Type in separator character, if no, leave it blank. Then, Upload your file. At this time, in your right-hand side, you will see Review your dataset panel opened with a brief review of your dataset, including all the variables and the first six observations. Make sure its the right dataset you want to analyse, and move to next step.
3. Use the select box to choose equivalent variable, given all the variables we read from your dataset. For Covariate you could choose either one or multiple into study.
4. Move the slider to choose a value for Sensitivity and Specificity, then Submit to get your analysis results.
